# Contributing to ETL Pipeline Project

Thank you for your interest in contributing to this project!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/etl-pipeline-project.git`
3. Create a virtual environment: `python -m venv venv`
4. Install dependencies: `pip install -r requirements.txt`
5. Copy `.env.example` to `.env` and configure your database
6. Setup database: `python scripts/setup_database.py`

## Development Workflow

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test your changes
4. Commit your changes: `git commit -m "Add your feature"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Create a Pull Request

## Code Style

- Follow PEP 8 Python style guide
- Add docstrings to functions and classes
- Include comments for complex logic
- Update README.md if adding new features

## Testing

Before submitting a PR, please:
- Test your changes locally
- Ensure all scripts run without errors
- Verify database operations work correctly

## Questions?

Feel free to open an issue for questions or suggestions!
